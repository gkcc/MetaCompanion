# 推荐模型设计

推荐层的目标是回答一个问题：在最近环境里，我现在更适合使用哪一种牌组形态。

## 数据层

- 远端环境：HSReplay Premium 的形态热度和形态对阵胜率。
- 本地环境：HDT `DeckStats.xml` 导出的近期对局，再用 HSReplay 牌组库匹配对手形态。
- 牌组库：`%AppData%\HearthstoneDeckTracker\MetaCompanion\hsreplay_deckcodes.txt`。`archetype_deck_branches.tsv` 仅作为旧数据兜底。

## 当前评分

第一版使用加权期望胜率：

```text
候选形态期望胜率 =
  sum(对手形态占比 * 候选形态对该形态胜率)
```

远端环境默认取：

- 标准模式天梯。
- 钻石到传说。
- 最近 3 天。
- 总热度前 20。
- 每职业前 5，去重后合并。

输出字段：

- `expected_win_rate`：按环境加权后的期望胜率。
- `coverage_pct`：有足够样本对阵数据覆盖了多少环境。
- `weighted_sample_games`：被采用对阵单元格的加权样本量。
- `fallback_win_rate`：样本不足时的回退胜率。

## 本地加权

个人推荐会把本地近期对手分布混入远端环境：

```text
最终环境权重 =
  远端环境权重 * (1 - 本地权重) +
  本地近期对手权重 * 本地权重
```

默认本地权重是 `0.35`，统计最近 3 天。这个数值偏保守，避免少量本地样本把网页端大样本完全盖掉。

## 后续方向

- 按补丁时间衰减旧对局。
- 对低置信度本地形态降低权重。
- 按分段、时间段、职业分布分别给权重。
- 在赛后浮窗中展示“推荐为什么是它”，方便人工判断可信度。
