# Meta Companion 一键脚本

这些 `.cmd` 文件可以直接双击运行，方便手动维护 Meta Companion。

- `01 构建Release版本.cmd`：编译 Release x86 DLL。
- `02 构建并安装插件到HDT（先关闭HDT）.cmd`：编译并安装插件；HDT 必须先关闭。
- `03 构建并等待HDT关闭后安装插件.cmd`：先编译，然后等 HDT 退出后自动安装。
- `04 手动刷新远程数据库（智能跳过）.cmd`：运行远程缓存刷新；如果今天已经刷新完整会跳过。
- `05 强制刷新远程数据库.cmd`：忽略“今天已刷新”的判断，强制重拉远程缓存。
- `06 安装自动刷新计划任务（管理员）.cmd`：安装每日 08:05 加登录后补跑的 Windows 计划任务，会弹管理员权限。
- `07 刷新本地记录和个人推荐.cmd`：刷新本地对局统计和个人推荐，不重拉完整远程库。
- `08 运行测试（沙箱不改HDT配置）.cmd`：运行测试；脚本会隔离 HDT AppData 并保护真实 `config.xml`。
- `09 发布前检查.cmd`：运行发布门禁。
- `10 标记当前补丁时间.cmd`：把当前时间写入补丁标记，用于个人推荐权重。
- `11 清理构建缓存.cmd`：清理 `bin/obj` 等构建缓存。

建议：

- 安装插件前先关闭 HDT，或者使用“等待HDT关闭后安装”。
- 手动刷新远程数据库通常用“智能跳过”；只有怀疑数据坏了或不够新时才用“强制刷新”。
- 计划任务安装需要管理员权限，这是 Windows 任务计划程序的限制。
